# Atova PDF Generation — Developer Guide (cleared and replaced)

This project will be handed to a ReactJS developer to convert Figma certificate designs into plain HTML/CSS for PDF generation inside a Laravel app. The PDF generator used in this project does not support CSS frameworks, so use raw CSS and minimal HTML.

Quick overview (what you must do)
- Clone the repo to your local machine.
- Use Laragon or XAMPP (or similar) and put the project in the webserver's projects folder.
- Open and edit resources/views/pdf/index.blade.php — treat it as a plain HTML file with inline or embedded CSS.
- Add fonts to public/fonts and reference them with @font-face using absolute file paths or Laravel helpers.

Step-by-step setup (for developers who do not know Laravel/PHP)
1. Prerequisites
   - Git
   - PHP 8.x
   - Composer
   - Laragon (recommended) or XAMPP
   - A code editor (VS Code recommended)

2. Clone repository
   - Open a terminal and run:
     git clone <repository-url> C:\laragon\www\atova-pdf-generation
   - (If you use XAMPP, you may clone into C:\xampp\htdocs\atova-pdf-generation instead.)

3. Install PHP dependencies
   - Open a terminal in the project folder:
     cd C:\laragon\www\atova-pdf-generation
     composer install

4. Environment file and key
   - Copy environment example:
     copy .env.example .env
   - Generate app key:
     composer exec php -r "require 'vendor/autoload.php';"
     php artisan key:generate
   - (If you cannot run artisan yet, just copy .env and proceed. Key may be generated by the maintainer.)

5. Configure webserver
   - Laragon:
     - Place project inside C:\laragon\www\ (already cloned there).
     - Laragon auto-detects projects. Start Laragon > Start All.
     - Open the site via http://localhost/atova-pdf-generation/public or set a pretty URL via Laragon's menu.
   - XAMPP:
     - Place project inside C:\xampp\htdocs\.
     - Start Apache from XAMPP Control Panel.
     - Visit http://localhost/atova-pdf-generation/public
   - Alternative: php artisan serve (development only)
     - php artisan serve --host=127.0.0.1 --port=8000
     - Visit http://127.0.0.1:8000

6. Where to work
   - Primary template: resources/views/pdf/index.blade.php
   - Treat this file as plain HTML. Blade syntax exists but you can ignore PHP parts; keep placeholders like {{ $name }} if present — you may keep them unchanged.
   - PDF generation is typically done via a controller route that loads this view and passes data. You do not need to modify controllers for HTML/CSS work.

HTML/CSS rules for PDF generation
- Do not use CSS frameworks (Bootstrap/Tailwind) — they may not render well in the PDF generator.
- Prefer embedded CSS: place a <style> ... </style> block at the top of resources/views/pdf/index.blade.php.
- Avoid external network fonts. Use local font files and @font-face.
- Keep layout simple with absolute sizes (mm/px) and avoid complex flex/grid behaviors that PDF engines may not support fully.
- Use inline images from public/ (e.g., <img src="{{ public_path('images/logo.png') }}"> or file paths). For PDF generators, absolute paths or inline base64 images are more reliable.

How to add extra fonts (simple and reliable)
1. Copy font files (.ttf, .otf, .woff) into:
   C:\laragon\www\atova-pdf-generation\public\fonts\
2. In resources/views/pdf/index.blade.php add an embedded style with @font-face. Example:

```html
<!-- filepath: resources/views/pdf/index.blade.php -->
<style>
/* Example: using an absolute file path (Windows) — replace username/path accordingly */
@font-face {
  font-family: 'MyCustomFont';
  src: url("file:///C:/laragon/www/atova-pdf-generation/public/fonts/MyCustomFont-Regular.ttf") format('truetype');
  font-weight: normal;
  font-style: normal;
}

/* Or use Laravel helper to print an absolute path (Blade can be used inside the style tag) */
@font-face {
  font-family: 'MyCustomFont2';
  src: url("{{ public_path('fonts/MyCustomFont2-Regular.ttf') }}") format('truetype');
  font-weight: normal;
  font-style: normal;
}

/* Basic certificate CSS example (use as starting point) */
body {
  margin: 0;
  padding: 0;
  font-family: 'MyCustomFont', sans-serif;
  background: #ffffff;
  color: #222;
}

/* Container sized for A4 landscape (adjust as needed) */
.certificate {
  width: 1122px; /* A4 landscape at 96 DPI ~ 1122px x 793px — adjust to your PDF generator DPI */
  height: 793px;
  padding: 60px;
  box-sizing: border-box;
  position: relative;
  border: 10px solid #e6e6e6;
  background: white url("{{ public_path('images/certificate-bg.png') }}") no-repeat center;
  background-size: cover;
}

.certificate .title {
  text-align: center;
  font-size: 36px;
  font-weight: 700;
  margin-top: 20px;
}

.certificate .recipient {
  text-align: center;
  margin-top: 40px;
  font-size: 48px;
  font-weight: 600;
  letter-spacing: 1px;
}

.certificate .meta {
  text-align: center;
  margin-top: 30px;
  font-size: 18px;
  color: #555;
}

.signature {
  position: absolute;
  right: 80px;
  bottom: 80px;
  text-align: center;
}

.signature img {
  width: 180px;
  height: auto;
}

/* Small helper for printed date */
.date {
  position: absolute;
  left: 80px;
  bottom: 80px;
  font-size: 16px;
}
</style>
```

Notes about font paths and Blade:
- Using file:/// absolute paths works reliably with DomPDF on Windows.
- Using {{ public_path('fonts/...') }} prints an absolute server path that DomPDF can use.
- Avoid using asset() inside @font-face because asset() produces a URL (http://localhost/...), which may fail if the PDF generator cannot fetch remote URLs. File system paths are more reliable.

Editing tips for the React developer (no Laravel required)
- Open resources/views/pdf/index.blade.php. Replace the HTML structure and CSS inside <style> to match the Figma design.
- Keep dynamic placeholders ({{ $name }}, {{ $date }}) if present — do not remove them unless asked.
- Use only raw CSS (no Tailwind classes, no Bootstrap preprocessor classes). Use simple selectors and explicit sizes.
- For images, put them into public/images/ and reference as shown above.
- If you need to test quickly without PHP, create a small static HTML file locally and test the layout, then copy CSS/HTML into the blade file.

Testing the PDF output (developer workflow)
- After editing index.blade.php, open the route that generates the PDF in the browser (ask the maintainer for the route URL or check routes/web.php for routes that reference the PDF view).
- If you do not have the route, you can ask the maintainer to provide a quick test route that returns the view directly (no PDF) so you can preview the design in the browser:
  return view('pdf.index', ['name' => 'Test User', 'date' => '2025-01-01']);
- Once HTML is correct, test the actual PDF generator (controller that uses the PDF package) and adjust CSS as needed.

Short CSS checklist for certificates
- Use embedded <style> at top of blade file.
- Use @font-face with file system paths.
- Use pixel or mm units for predictable sizing.
- Avoid external CSS frameworks and remote fonts.
- Use images from public/ and reference via absolute path or public_path helper.

If you need help
- Provide the maintainer with:
  - The edited resources/views/pdf/index.blade.php file.
  - Any new images placed in public/images/.
  - Any new fonts placed in public/fonts/.

This README is focused on the developer task: converting Figma to raw HTML/CSS inside Laravel blade for PDF generation. Keep edits minimal and focused on the view file; do not modify backend PDF logic without coordination.
